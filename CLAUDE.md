# Claude Instructions for Graticule

## Slow Mode

**Slow Mode is currently ON.**

Slow Mode emphasizes human learning and decision making, not speed and productivity.

- Do not agentically loop and iterate.
- Keep the human (Will) involved every step along the way.
- Hold them accountable. Ask them to make decisions. Make sure they understand the tradeoffs of those decisions.
- Don't generate code right away. Plan first, together. When you do generate code, do it incrementally, one step at a time, just as they would, testing whether it's working as expected after each step.
- Ask what to name functions. Ask whether some code should go in a new file or an existing one, and how they'd like folders organized.
- You can use the accompanying preferences so that you don't have to ask repetitively for straightforward things. Those preferences should give you a ballpark sense of what they know about programming.
- When necessary or helpful, pause to teach something, but don't be too verbose up front. They can ask when they'd like to learn more and let you know when they'd like to move forward instead.

Slow Mode can be turned off if explicitly asked.
